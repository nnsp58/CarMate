# Plan: Build Release APK for RideOn Flutter App

## Objective
Generate the release APK file for the RideOn carpooling app, resulting in an APK approximately 65.9 MB in size.

## Prerequisites
- Flutter SDK installed and configured
- Android SDK with build tools installed
- Keystore configured for release signing (in android/key.properties and keystore file)
- All dependencies installed (run `flutter pub get`)

## Steps
1. **Navigate to project root**: Ensure we're in `D:\rideon-1\rideon`
2. **Clean previous builds**: Run `flutter clean` to remove old build artifacts
3. **Build release APK**: Execute `flutter build apk --release`
4. **Locate APK file**: Find the generated APK at `build/app/outputs/flutter-apk/app-release.apk`
5. **Verify size**: Confirm the APK size is around 65.9 MB

## Fallback Android Build Method (if Flutter build fails)
If the Flutter build command fails, use the Android Gradle wrapper:
1. Navigate to `android/` directory
2. Run `.\gradlew.bat assembleRelease`
3. Locate APK at `android/app/build/outputs/apk/release/app-release.apk`

## Validation
- Build completes without errors
- APK file exists and is properly signed
- App installs and runs on test device

## Notes
- The `bundleRelease` command you ran generates an AAB (Android App Bundle), not APK. Use `assembleRelease` for APK.
- Ensure minSdkVersion and targetSdkVersion are set correctly in android/app/build.gradle</content>
